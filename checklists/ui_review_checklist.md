# UI Review Checklist

Use this checklist to keep the app practical, structured, and consistent.

## Navigation

- [ ] Left navigation exists where appropriate.
- [ ] Sidebar can collapse and expand.
- [ ] Main sections are clear.
- [ ] Sub-sections are grouped logically.
- [ ] Expandable nav groups are used where useful.
- [ ] Active page is visible.
- [ ] Icons support labels but do not replace labels.

## Page structure

- [ ] Page has a clear heading.
- [ ] Page has useful subheadings.
- [ ] Large areas are split into collapsible sections where useful.
- [ ] Primary action is obvious.
- [ ] Secondary actions are not visually confusing.

## Data display

- [ ] Main working data is shown in rows and columns.
- [ ] Data tables support quick search.
- [ ] Data tables support quick filters.
- [ ] Data tables support sorting.
- [ ] Important columns are visible by default.
- [ ] Large datasets have pagination or virtualisation where needed.

## Drill-down and detail views

- [ ] Row click opens a detail popup, drawer, or page where useful.
- [ ] Detail view shows grouped information.
- [ ] Detail view has tabs where useful.
- [ ] Related records are visible where useful.
- [ ] Audit/history is visible where useful.

## Cards and dashboards

- [ ] Cards are used for summaries only.
- [ ] Cards are not used for data that belongs in tables.
- [ ] Dashboards answer management questions.
- [ ] Charts use clear labels.
- [ ] Charts are not decorative only.

## Forms

- [ ] Forms use SQL lookup dropdowns where practical.
- [ ] Required fields are visible.
- [ ] Validation messages are useful.
- [ ] Save/cancel/reset actions are clear.
- [ ] Date pickers are reusable.
- [ ] The same form pattern is reused across pages.

## Zoom and accessibility

- [ ] Zoom controls exist where useful.
- [ ] Zoom works in 10% increments.
- [ ] Reset to 100% is available.
- [ ] Text remains readable.
- [ ] Light/dark mode remains readable where included.
- [ ] Status colours have text labels or clear meaning.

## Final UI review

- [ ] App does not look like a generic toy dashboard.
- [ ] App supports operational work.
- [ ] Hierarchy and drill-down are clear.
- [ ] No duplicate UI patterns were created unnecessarily.
